# Frontend UX Mobile Skill

Reusable checklist for improving WB Studio front-end quality, visual hierarchy, responsive behavior, and mobile user experience.

## Primary Goal

- Make the website feel premium, minimal, fast, and credible on mobile first.
- Support private house and villa clients with clear messaging, elegant presentation, and a frictionless path to contact.

## UX Priorities

- Create a clear hero message that explains the offer quickly.
- Make the primary CTA obvious in the first screen.
- Build a strong above-the-fold experience with visible credibility and next steps.
- Use readable spacing, calm navigation, and a direct contact path.
- Keep sections focused so visitors can scan without confusion.

## Mobile Priorities

- Use thumb-friendly buttons and tap targets.
- Keep typography readable without zooming.
- Prevent horizontal overflow on all screen sizes.
- Favor fast loading, simple forms, short sections, and clear above-the-fold intent.
- Make WhatsApp, Book Free Call, or Plot Intake easy to reach from key moments.

## Visual Priorities

- Aim for a premium architectural feel with disciplined whitespace and consistent rhythm.
- Use strong image selection that supports private villa, house, and residence positioning.
- Avoid a generic template look by improving hierarchy, proportion, layout, and detail quality.
- Keep visual choices restrained, intentional, and conversion-focused.

## Branding Freedom

- Colors, typography, spacing, and layout may be improved when the new direction feels more premium, credible, and conversion-focused.
- Do not blindly preserve the existing palette if a stronger Saudi/GCC-facing visual direction is available.

## Accessibility Basics

- Maintain strong text contrast and visible focus states.
- Use semantic structure with logical headings and landmarks.
- Support useful alt text for meaningful images.
- Respect reduced motion preferences and avoid motion that blocks comprehension.

## Avoid

- Heavy animations, clutter, decorative noise, low-quality stock imagery, and unnecessary frameworks.
- Visual gimmicks that slow the site, distract from the offer, or weaken trust.

## Self-Update Rule

- This skill may update itself when website changes, new assets, new media, new pages, UX findings, or durable design decisions make an update valuable. Keep updates concise and high-impact. Do not update the skill for temporary ideas, minor wording preferences, low-value observations, private data, credentials, or noise.
