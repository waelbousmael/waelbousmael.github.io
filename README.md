# WB Studio ‚Äî wbstudio.xyz

Bilingual (Arabic / English) portfolio website for **WB Studio**, the personal
architecture practice of **Wael Bousmael** ‚Äî luxury private villas, interiors,
and spatial concepts in Jeddah, Saudi Arabia, and the Gulf.

Static site, hosted on **GitHub Pages** (custom domain via `CNAME`). No
framework, no dependencies ‚Äî pages are generated by a single zero-dependency
Node script.

## How it works

```
data/projects.json      ‚Üê portfolio projects (bilingual) ‚Äî EDIT THIS
data/site.mjs           ‚Üê all other site copy + config (bilingual) ‚Äî EDIT THIS
scripts/build.mjs       ‚Üê generator: templates + sitemap
assets/css/site.css     ‚Üê the whole design system
assets/js/site.js       ‚Üê reveal animation, mobile menu, language memory
assets/js/contact.js    ‚Üê lead form (FormSubmit) + dormant Google Maps hook
assets/projects/<slug>/ ‚Üê images per project (cover.webp, gallery-NN.webp)
assets/projects/_incoming/  ‚Üê staging folder for your real work (see its README)
assets/licenses/LICENSES.md ‚Üê license register for every external asset
```

Generated output (do **not** edit by hand ‚Äî files carry a banner comment):

- Arabic (default, RTL) at the root: `/`, `/work/`, `/work/<slug>/`,
  `/services/`, `/<landing-slug>/`, `/approach/`, `/journal/`,
  `/journal/<article-slug>/`, `/about/`, `/contact/`, `/credits/`
- English (LTR) mirrored under `/en/...`
- `sitemap.xml` with `hreflang` alternates

### SEO landing pages & journal (data-driven)

High-intent landing pages (e.g. `villa-design-jeddah`, `obhur-villa-design`)
live in the `landings` array in `data/site.mjs`; SEO guides live in the
`journal` array. Each is bilingual `{ ar, en }` and auto-generates its page,
nav/footer links, internal links, `sitemap.xml` entry, and schema:

- **landings** ‚Üí `Service` + `FAQPage` + `BreadcrumbList` (each must target a
  *distinct* search intent ‚Äî do not duplicate copy between pages).
- **journal** ‚Üí `Article` + `BreadcrumbList`.

To add one, copy an existing entry in the relevant array, give it a unique
`slug`, write genuinely distinct bilingual copy, then run the build. Link new
landings to related projects via `relatedProjects` (project slugs) and to
sibling pages via `related` (landing slugs).

## Run / build

```bash
node scripts/build.mjs        # regenerate all pages + sitemap
python -m http.server 8123    # preview at http://localhost:8123
```

Requires only Node ‚â• 18. GitHub Pages serves the committed HTML as-is; run the
build locally and commit the output.

## Add a project

1. Create `assets/projects/<slug>/` with `cover.webp` (~1800‚Äì2000px, 16:10) and
   `gallery-01.webp ‚Ä¶` (~1400‚Äì1600px landscape / ~1100‚Äì1200px portrait).
2. Add an entry to `data/projects.json` (copy an existing one). Every text
   field needs both `ar` and `en`. Record real image `w`/`h` values.
3. If the images are not yours, add them to `assets/licenses/LICENSES.md`;
   if they are yours, set `"placeholder": false`.
4. `node scripts/build.mjs` ‚Üí review ‚Üí commit (including the generated HTML).

## Remove / reorder projects

Delete (or move) the entry in `data/projects.json`, delete the image folder,
rebuild. Order on the Work page = array order. Homepage shows the first three
projects with `"featured": true` (the first renders full-width).

## Replace placeholder images with your own

See `assets/projects/_incoming/README.md` ‚Äî short version: drop your files in,
swap paths in `data/projects.json`, set `placeholder: false`, update
`LICENSES.md`, rebuild.

## Bilingual content rules

- Arabic is the default language and lives at the root with `dir="rtl"`.
- English mirrors every page under `/en/` with `dir="ltr"`.
- The header toggle links to the same page in the other language and remembers
  the choice in `localStorage` (`wb_lang`). There is no JS dictionary anymore ‚Äî
  each page is fully prerendered in its language (better SEO: both languages
  are crawlable, with `hreflang` annotations).
- Never hardcode visible text in `scripts/build.mjs` ‚Äî put it in `data/site.mjs`
  or `data/projects.json` as `{ ar, en }`.
