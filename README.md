# WB Studio — wbstudio.xyz

Bilingual (Arabic / English) portfolio website for **WB Studio**, the personal
architecture practice of **Wael Bousmael** — luxury private villas, interiors,
and spatial concepts in Jeddah, Saudi Arabia, and the Gulf.

Static site, hosted on **GitHub Pages** (custom domain via `CNAME`). No
framework, no dependencies — pages are generated by a single zero-dependency
Node script.

## How it works

```
data/projects.json      ← portfolio projects (bilingual) — EDIT THIS
data/site.mjs           ← all other site copy + config (bilingual) — EDIT THIS
scripts/build.mjs       ← generator: templates + sitemap
assets/css/site.css     ← the whole design system
assets/js/site.js       ← reveal animation, mobile menu, language memory
assets/js/contact.js    ← lead form (FormSubmit) + dormant Google Maps hook
assets/projects/<slug>/ ← images per project (cover.webp, gallery-NN.webp)
assets/projects/_incoming/  ← staging folder for your real work (see its README)
assets/licenses/LICENSES.md ← license register for every external asset
```

Generated output (do **not** edit by hand — files carry a banner comment):

- Arabic (default, RTL) at the root: `/`, `/work/`, `/work/<slug>/`,
  `/approach/`, `/about/`, `/contact/`, `/credits/`
- English (LTR) mirrored under `/en/...`
- `sitemap.xml` with `hreflang` alternates

## Run / build

```bash
node scripts/build.mjs        # regenerate all pages + sitemap
python -m http.server 8123    # preview at http://localhost:8123
```

Requires only Node ≥ 18. GitHub Pages serves the committed HTML as-is; run the
build locally and commit the output.

## Add a project

1. Create `assets/projects/<slug>/` with `cover.webp` (~1800–2000px, 16:10) and
   `gallery-01.webp …` (~1400–1600px landscape / ~1100–1200px portrait).
2. Add an entry to `data/projects.json` (copy an existing one). Every text
   field needs both `ar` and `en`. Record real image `w`/`h` values.
3. If the images are not yours, add them to `assets/licenses/LICENSES.md`;
   if they are yours, set `"placeholder": false`.
4. `node scripts/build.mjs` → review → commit (including the generated HTML).

## Remove / reorder projects

Delete (or move) the entry in `data/projects.json`, delete the image folder,
rebuild. Order on the Work page = array order. Homepage shows the first three
projects with `"featured": true` (the first renders full-width).

## Replace placeholder images with your own

See `assets/projects/_incoming/README.md` — short version: drop your files in,
swap paths in `data/projects.json`, set `placeholder: false`, update
`LICENSES.md`, rebuild.

## Bilingual content rules

- Arabic is the default language and lives at the root with `dir="rtl"`.
- English mirrors every page under `/en/` with `dir="ltr"`.
- The header toggle links to the same page in the other language and remembers
  the choice in `localStorage` (`wb_lang`). There is no JS dictionary anymore —
  each page is fully prerendered in its language (better SEO: both languages
  are crawlable, with `hreflang` annotations).
- Never hardcode visible text in `scripts/build.mjs` — put it in `data/site.mjs`
  or `data/projects.json` as `{ ar, en }`.
